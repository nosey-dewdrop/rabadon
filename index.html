in development · building in public

it knows what is going wrong through your code, so you can touch the grass.

rabadon runs your pipeline and checks itself while it runs. a step breaks silently, it catches it before it spreads, and repairs it. no babysitting.

i started building it to solve my own problem, after shipping 20+ projects where the plumbing broke quietly. it is not a finished product yet. the core runs; the rest is being built in the open.

bound by construction

a loop that can run away won't compile.

the pipeline refuses to start without a bound. no depth, no token ceiling, no run. the runaway that burns tokens can't be written by accident.

bound
// no bound -> .run() refuses rabadon.pipeline() .step('draft', draft) .run(input) // Error: no .bound() set. // an unbounded pipeline is a // runaway waiting to happen.

caught before it spreads

a step that silently drops a feature is caught at the step, not eight steps later.

each step's output is checked against what you meant, before it flows onward. the drop that used to surface as a wrong answer downstream is named where it was born.

correct[]
step('normalize', normalize, { correct: [ named('noRecordVanishes', (out, inp) => out.length === inp.length ? true : 'id 0 vanished silently') ] })

diagnose → stop → repair

it doesn't just catch the break. it walks the pipeline back to a working one.

when a step fails its check, rabadon hands the broken output and the exact reasons to a repair, re-checks the fix, and only accepts it if it passes. bounded, so the fixer can't run away either.

repair
step('normalize', normalize, { correct: [noRecordVanishes], repair: (broken, inp, fails) => // this step produced `broken`, // checks failed for `fails`, // rewrite, then re-check. fixNormalize(inp) }) // verdict: PASS (ada kept)

where it is right now

day 1

the core is the only part that exists so far: the four ideas (bound, catch, repair, don't-drop-silently) are written as one small primitive, and it has a test file that runs. that is the whole of it today. no product, no users, no live service.

the plan is honest and public: wire the repair step to a real model, hook it onto a live pipeline instead of a demo, and only then publish a benchmark number worth trusting. proof will come as it is built, not before. follow it on github.

stop babysitting the plumbing.

rabadon is the layer that holds the pieces together so the pipeline stays true to its purpose without you watching it. it is being built in the open, one honest step at a time. the core is here; follow along.
